# Codex 使用简介

## 我是谁

我是 Codex，一个面向代码与工程任务的 AI 助手，可以直接在当前项目环境中协助你完成开发工作。

我更适合做这些事情：

- 阅读和理解项目代码结构
- 编写、修改、重构代码
- 排查报错和定位问题
- 补充脚本、文档和测试
- 解释实现逻辑并给出可执行建议

## 主要用法

你可以直接用自然语言给我下达任务，常见方式如下：

### 1. 让我要代码

示例：

- 帮我给这个项目加一个配置文件加载模块
- 在 `run_pipeline.py` 里增加命令行参数
- 给这个接口补一个单元测试

### 2. 让我查问题

示例：

- 这个报错是什么意思，帮我定位原因
- 为什么这个脚本运行很慢
- 帮我检查这段代码有没有明显 bug

### 3. 让我读代码并解释

示例：

- 帮我解释这个项目的执行流程
- `modules/` 目录主要负责什么
- 这段函数的输入输出分别是什么

### 4. 让我写文档或整理说明

示例：

- 帮我写一个 README 的快速开始部分
- 把这个功能整理成使用说明
- 给这个模块补一份中文介绍文档

## 推荐提问方式

为了让我更快给出可用结果，建议你在提问时尽量带上这些信息：

- 目标是什么
- 希望修改哪些文件或模块
- 是否有报错信息
- 是否有期望输出格式

例如：

> 帮我在 `run_traffic_validation.py` 里新增一个参数 `--sample-size`，默认值为 20，并同步更新说明文档。

## 使用特点

- 我会先查看项目上下文，再动手修改
- 能直接在仓库里创建或编辑文件
- 会尽量给出可验证的结果
- 如果信息不足，会基于当前代码做合理判断

## 注意事项

- 我依赖你当前提供的项目和上下文
- 涉及运行环境、外部服务或依赖时，结果可能受本地条件影响
- 对高风险改动，最好明确说明你的预期

## 一句话总结

你可以把我当成一个能读代码、改代码、查问题、写说明的工程助手，直接告诉我任务即可。
